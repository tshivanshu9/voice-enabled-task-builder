# üé§ Voice-Enabled Task Tracker

A modern task management application that allows users to create tasks using voice commands. Speak naturally and let AI parse your voice input into structured task data.

## üöÄ Project Setup

### Prerequisites

- **Node.js**: Version 22 required
- **MongoDB**: Local installation or MongoDB Atlas account
- **Google AI Studio**: API key for Gemini model
- **Modern Browser**: Chrome, Firefox, Safari (for Speech Recognition API)

### Installation Steps

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd voice-todo
   ```

2. **Install root dependencies**
   ```bash
   npm install
   ```

3. **Backend Setup**
   ```bash
   cd backend
   npm install
   ```

4. **Frontend Setup**
   ```bash
   cd ../frontend
   npm install
   ```

### Environment Configuration

Create `.env` file in the `backend` directory:

```env
# Database
MONGO_URI=mongodb://localhost:27017/voice-todo
# or for MongoDB Atlas:
# MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/voice-todo

# Server
PORT=8000

# AI Service
GEMINI_API_KEY=your_gemini_api_key_here
```

### Running the Application

1. **Start both frontend and backend** (from root directory):
   ```bash
   npm run both
   ```

2. **Or run individually:**
   ```bash
   # Backend only
   npm run backend

   # Frontend only
   npm run frontend
   ```

3. **Access the application:**
   - Frontend: http://localhost:5173
   - Backend API: http://localhost:8000

### Seed Data

No initial seed data required. The application works with an empty database.

### Tasks Endpoints

#### GET /tasks
Get all tasks with pagination

**Query Parameters:**
- `page` (optional): Page number (default: 1)
- `limit` (optional): Items per page (default: 10)

**Response:**
```json
{
  "success": true,
  "data": [
    {
      "_id": "507f1f77bcf86cd799439011",
      "title": "Buy groceries",
      "description": "Remember to buy milk and bread",
      "dueDate": "2024-12-10",
      "priority": "medium",
      "taskStatus": "todo",
      "createdAt": "2024-12-06T10:00:00.000Z"
    }
  ],
  "paginate": {
    "totalCount": 15,
    "page": 1,
    "limit": 10
  }
}
```

#### GET /tasks/:id
Get a specific task

**Response:**
```json
{
  "success": true,
  "data": {
    "_id": "507f1f77bcf86cd799439011",
    "title": "Buy groceries",
    "description": "Remember to buy milk and bread",
    "dueDate": "2024-12-10",
    "priority": "medium",
    "taskStatus": "todo"
  }
}
```

#### POST /tasks
Create a new task

**Request Body:**
```json
{
  "title": "Buy groceries",
  "description": "Remember to buy milk and bread",
  "dueDate": "2024-12-10",
  "priority": "medium",
  "taskStatus": "todo"
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "_id": "507f1f77bcf86cd799439011",
    "title": "Buy groceries",
    "description": "Remember to buy milk and bread",
    "dueDate": "2024-12-10",
    "priority": "medium",
    "taskStatus": "todo",
    "createdAt": "2024-12-06T10:00:00.000Z"
  }

```

#### PUT /tasks/:id
Update a task

**Request Body:** (partial update supported)
```json
{
  "title": "Updated title",
  "taskStatus": "in-progress"
}
```

#### DELETE /tasks/:id
Delete a task

**Response:**
```json
{
  "success": true,
  "message": "Task deleted successfully"
}
```

### Parse Endpoint

#### POST /parse
Parse natural language text into task structure

**Request Body:**
```json
{
```

**Response:**

## üéØ Decisions & Assumptions

### Key Design Decisions

1. **Monorepo Structure**: Organized frontend and backend in a single repository for easier development and deployment.

2. **RESTful API Design**: Standard REST endpoints with clear HTTP methods and status codes.

3. **Soft Delete**: Tasks are marked as inactive rather than permanently deleted for data recovery.

4. **AI-First Parsing**: Natural language input is processed through Gemini AI for intelligent task extraction.

5. **Browser Speech Recognition**: Using native Web Speech API instead of external services for better privacy and reduced latency.

### Data Models

**Task Model:**
- `title`: Short, descriptive task name
- `description`: Detailed task information
- `dueDate`: Optional due date in YYYY-MM-DD format
- `priority`: low/medium/high or null
- `taskStatus`: todo/in-progress/done
- `status`: active/inactive (for soft delete)

### Assumptions Made

1. **Browser Compatibility**: Assumes modern browsers with Speech Recognition API support.

2. **Language**: English-only speech recognition and AI parsing.

3. **Date Parsing**: Relative dates ("tomorrow", "next Friday") calculated based on current date.

4. **Single User**: No user authentication or multi-tenancy implemented.

5. **Task Scope**: Tasks are general-purpose without specific categorization or project grouping.

6. **AI Reliability**: Assumes Gemini AI provides consistent JSON responses for task parsing.
